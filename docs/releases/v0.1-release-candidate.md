# VibeBoard Runtime v0.1 Release Candidate

Date: 2026-07-08

This release candidate closes the current Runtime productization pass. Treat it
as the repeatable v0.1 baseline for the LCKFB/Lichuang ESP32-S3 board, while
keeping the post-v0.1 backlog for physical QA, long soaks, cloud providers, and
more upstream apps.

## Scope

The release-candidate scope includes:

- ESP-IDF Runtime firmware for the 320x240 LCKFB/Lichuang ESP32-S3 board.
- SD-card Lua/LVGL app install, package manifest v2 validation, launch, stop,
  rescan, delete, and runtime config writes.
- Native launcher, HTTP API, and local Device Web UI management surface.
- Runtime Lua modules for app lifecycle, file, network, JSON/time, input,
  touch, gamepad, IMU, camera, I2S, system status, and NES native facade.
- Camera/Photos loop with live preview, capture to SD, camera gallery, and
  standalone Photos display.
- Voice AI bridge path with board recording, desktop bridge upload, local STT,
  and returned text display.
- Weather resource-loading architecture with packager-generated `.vbimg`
  backgrounds, PSRAM prefetch/cache, and 32-bit BMP alpha icons.
- New-app documentation and package resource metadata for future generated apps.

## Board Evidence Already Recorded

Recent board evidence is recorded in `docs/device-bringup.md`,
`docs/runtime-capabilities.md`, and `docs/next-session-plan.md`. Highlights:

- Runtime boots on the board, mounts SD, starts WiFi, serves HTTP on port 8080,
  and exposes launcher/device status.
- Weather starts with `.vbimg` background prefetch/cache and 32-bit alpha BMP
  icons; the user observed the launch path as much faster.
- Camera live preview, capture, SD photo save, camera gallery, and standalone
  Photos display were user-verified.
- Voice AI records audio, uploads to the desktop bridge, receives local STT
  output, and displays the returned text.
- I2S speaker tone and record/playback diagnostics have audible/user evidence.
- NES legal-ROM smoke, synthetic gamepad host path, and host-audio backend have
  machine evidence, with long sustained performance left for follow-up.

## Not Claimed Complete

These items are explicitly post-v0.1:

- Full physical visual QA sweep across every migrated app.
- Real BLE/Xbox controller discovery and long gameplay validation.
- Cloud STT/LLM provider productization, credentials, privacy logging policy,
  and weak-network failure UX.
- Long-running soak tests across Weather, Camera/Photos, Voice AI, Spectrum,
  Videos, and NES.
- Device Web UI product polish such as browser-side package upload UX and
  richer log/error views.
- More upstream app migrations and broader HoloCubic API parity.
- Continued performance work for animation smoothness and heavy resource apps.
