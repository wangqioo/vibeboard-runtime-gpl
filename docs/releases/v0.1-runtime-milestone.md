# VibeBoard Runtime v0.1 Milestone

Date: 2026-06-29

This milestone closes the first machine-verifiable Runtime phase. It should be treated as a working development baseline, not a finished consumer product.

## Scope

The `v0.1-runtime-milestone` scope is:

- ESP32-S3 Runtime firmware for the LCKFB/Lichuang 320x240 board.
- SD-card Lua/LVGL app packages with `app.info`, assets, and Runtime capability checks.
- Native launcher, HTTP install/runtime API, and local Device Web UI.
- Curated migrated apps and smoke apps that can be packaged, uploaded, launched, stopped, and rescanned without reflashing firmware.
- NES native facade, Voice AI bridge path, I2S RX/TX smoke path, input smoke hooks, and app-manager lifecycle APIs.
- Documentation that separates machine evidence from physical, external-service, and long-soak gaps.

## Not Claimed Complete

The following are deliberately outside the v0.1 completion claim:

- Physical visual QA for migrated apps, including fonts, layout, media rendering, animations, and photos/videos/weather states.
- Real ES8311 amplifier and speaker listening tests.
- Real BLE/Xbox controller discovery and gameplay input.
- Real Voice AI STT provider success with production credentials.
- Real WiFi credential write-then-reboot verification with a fresh local credentials file.
- Long-running NES real-game soak and broader display/audio stress tests.
- Full upstream app parity and complete upstream WebUI/static asset support.
- Systematic performance work for migrated app startup and animation stutter.
