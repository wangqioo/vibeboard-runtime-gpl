# Runtime Capabilities

更新时间：2026-06-20

This document separates implemented API, build verification, and board verification. An API is not treated as fully done until it has a real device log in `docs/device-bringup.md`.

## Status Legend

| Status | Meaning |
| --- | --- |
| `board-verified` | Ran on the LCKFB ESP32-S3 board with serial, HTTP, or manual screen evidence. |
| `build-verified` | Compiles in ESP-IDF and has static/package tests, but has not been smoke-tested on board yet. |
| `planned` | Documented target, not implemented yet. |

## Planned Runtime Modules

| Capability | Target APIs | Status | Notes |
| --- | --- | --- | --- |
| Lua App manager | `app.list`, `app.rescan`, `app.current`, `app.exiting`, `app.exit`, `app.on("exit", cb)` | `build-verified lifecycle slice` | Runtime registers a Lua `app` module in each app runner. `app.list()` and `app.rescan()` scan `/sdcard/apps` through the same registry as Launcher/HTTP and return the latest compatible metadata table or `nil, err`; `app.current()` exposes current runner state and last result, `app.exiting()` mirrors the runner stop flag, `app.exit()` requests the active runner to stop from inside Lua, and `app.on("exit", cb)` fires during Lua cleanup. `apps/smoke_app_manager` is included in demo packaging for board smoke. | Input | `key.on`, `gamepad.state`, `gamepad.start`, `gamepad.stop`, `gamepad.on`, `gamepad.off`, `gamepad.rescan`, `gamepad.push_state` | `build-verified gamepad compatibility slice` | First touch-swipe-to-`key.on` bridge is board-verified through `2048`.
